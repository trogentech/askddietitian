# Dietrify Quiz System - Complete Understanding

## Overview
The Dietrify quiz is a sophisticated health assessment tool that uses dynamic question flow and rule-based personalization to provide tailored dietary recommendations based on user responses.

## 1. Quiz Session Flow & Progression

### State Management
- **currentStepIndex**: Tracks position in filtered steps array
- **answers**: Object storing all user responses with string keys matching question IDs
- **quizResult**: Stores the final analyzed result
- **isLoading**: Controls UI loading states during submission

### Progression Logic
```typescript
// Dynamic step filtering based on user answers
const filteredSteps = getFilteredSteps(answers);

// Navigation with validation
const canProceed = validateStep(currentStep, answers);
const shouldShowSubmit = isLastStep || isFollowupLastStep;
```

### Conditional Flow System
The quiz employs **multi-level conditional logic**:

1. **Step Level**: Entire sections (diabetes, pregnancy, PCOS, blood pressure) show/hide based on answers
2. **Question Level**: Individual questions appear based on previous responses  
3. **Option Level**: Answer choices can be conditionally displayed

Example progression flow:
```
Start → Basic Info → Weight Status → Conditions?
  ↓                ↓                 ↓
[Yes: Diabetes] → Diabetes Type → Diabetes Duration
[Yes: Pregnancy] → Pregnancy Trimester → Pregnancy Considerations
[Yes: PCOS] → PCOS Symptoms → PCOS Treatment
[No Conditions] → Diet Preferences → Activity Level → Results
```

## 2. Question Structure & Management

### Question Interface
```typescript
interface QuizQuestion {
  id: string;                    // Unique identifier for answer storage
  question: string;              // Question text displayed to user
  type: 'single' | 'multiple';   // Radio buttons or checkboxes
  options: QuizOption[];         // Available answer choices
  required?: boolean;            // Whether user must answer
  condition?: {                  // Conditional display logic
    field: string;
    value: string | string[];
  } | ((answers: QuizAnswers) => boolean);  // Function-based conditions
}
```

### Answer Options Structure
```typescript
interface QuizOption {
  id: string;                    // Option identifier
  text: string;                  // Display text for user
  value: string;                 // Stored value in answers object
  condition?: {                  // Option-specific conditions
    field: string;
    value: string;
  };
}
```

### Advanced Features
- **Function-based conditions**: Complex logic using full answers object
- **Multi-select support**: Checkbox questions with array handling
- **Dynamic validation**: Required field checking per question type
- **Conditional options**: Individual choices can appear/disappear

### Analysis Process
1. **Exact Match**: System searches for rules matching all user factors
2. **Fallback Logic**: If no exact match, uses age/gender-based defaults
3. **Priority Assignment**: Results classified by health urgency
4. **Risk Factor Identification**: Flags specific health concerns
5. **Personalization**: Tailors recommendations to user profile

### Fallback System
When no specific rule matches:
- **Age-based customizations**: Youth (18-39), Adult (40-64), Senior (65+)
- **Gender-specific advice**: Male vs. Female considerations
- **Pregnancy modifications**: Special recommendations for pregnant users
- **General wellness**: Basic healthy eating guidelines

### Answer Handling Logic
```typescript
const handleAnswerChange = (questionId: string, value: string, isMultiple = false) => {
  setAnswers((prev) => {
    if (isMultiple) {
      // Handle checkbox arrays - toggle selection
      const currentValues = (prev[questionId] as string[]) || [];
      const newValues = currentValues.includes(value)
        ? currentValues.filter((v) => v !== value)  // Remove if already selected
        : [...currentValues, value];                 // Add if not selected
      return { ...prev, [questionId]: newValues };
    }
    // Handle single selection (radio buttons)
    return { ...prev, [questionId]: value };
  });
};
```

### Navigation Logic
```typescript
const handleNext = () => {
  if (!validateStep(currentStep, answers)) {
    // Show validation error
    return;
  }
  
  if (shouldShowSubmit) {
    handleSubmit();
  } else {
    setCurrentStepIndex((prev) => prev + 1);
  }
};

const handlePrevious = () => {
  setCurrentStepIndex((prev) => Math.max(0, prev - 1));
};
```

### Conditional Filtering Function
```typescript
const getFilteredSteps = (answers: QuizAnswers): QuizStep[] => {
  return quizSteps.filter(step => {
    // Check step-level conditions
    if (step.condition) {
      if (typeof step.condition === 'function') {
        return step.condition(answers);
      }
      // Object-based condition checking
      const stepAnswer = answers[step.condition.field];
      return Array.isArray(step.condition.value) 
        ? step.condition.value.includes(stepAnswer)
        : stepAnswer === step.condition.value;
    }
    return true; // No condition means always show
  });
};
```

## 6. Data Flow Summary

1. **Initialization**: Quiz starts with first step, empty answers object
2. **Question Answering**: User selects options, stored in answers state
3. **Dynamic Filtering**: Each answer updates which future steps/questions appear
4. **Navigation**: System validates required fields before allowing progression
5. **Completion**: When all applicable questions answered, triggers result analysis
6. **Analysis Engine**: Matches user profile against 100+ recommendation rules
7. **Result Display**: Shows personalized recommendations with priority classification
8. **Email Delivery**: Optional detailed report sent via EmailJS
